# Subscribe & Daily Digest — Design Plan

## Overview

A subscribe button in the masthead collects email addresses. Every morning, active subscribers receive an email containing the daily briefing — the same editorial content generated by the existing `generate_briefing` pipeline.

The feature adds **two database tables**, **one new scheduled job**, **one new frontend component**, and an **email provider abstraction** that supports either Resend or Gmail SMTP.

## Email Provider

The worker sends email through an abstraction layer — `email/sender.py` — that dispatches to whichever provider is configured. This follows the same pattern as `llm/registry.py`, where config determines which backend is used.

**Provider selection** (checked in order):
1. If `RESEND_API_KEY` is set → use Resend
2. If `GMAIL_USER` and `GMAIL_APP_PASSWORD` are set → use Gmail SMTP
3. If neither → no-op (digest worker exits silently, same as `SLACK_WEBHOOK_URL` pattern)

The worker never imports provider-specific code directly. It calls `send_email()` and the abstraction handles the rest.

### Option A: Resend

| | |
|---|---|
| **Dependency** | `resend==2.7.0` (pip) |
| **Free tier** | 3,000/month, 100/day |
| **Paid tier** | $20/month for 50,000/month |
| **Setup** | Sign up at resend.com → get API key → add DNS records for custom domain |
| **From address** | Any address on a verified domain (e.g., `digest@yourdomain.com`) |
| **Deliverability** | Built on AWS SES. DKIM/SPF managed via Resend dashboard. |
| **SDK** | `resend.Emails.send({from_, to, subject, html, headers})` — synchronous HTTP call |
| **List-Unsubscribe** | Native header support |
| **Trade-off** | Requires a domain you control. Sends from `onboarding@resend.dev` until domain verified. |

**When to choose**: You own a domain and want professional `From` addresses, or plan to scale past 100 subscribers.

### Option B: Gmail SMTP

| | |
|---|---|
| **Dependency** | `aiosmtplib==3.0.2` (pip) |
| **Free tier** | 500 emails/day (free Gmail) or 2,000/day (Google Workspace) |
| **Paid tier** | Google Workspace starts at $7/user/month (you probably already have it) |
| **Setup** | Enable 2FA on Google account → generate App Password → done |
| **From address** | Must be the authenticated Gmail address (e.g., `yourname@gmail.com`) or a configured send-as alias |
| **Deliverability** | Google's infrastructure has high reputation. SPF/DKIM handled automatically for @gmail.com |
| **SDK** | `aiosmtplib` — async SMTP client, native to asyncio (no event loop blocking) |
| **List-Unsubscribe** | Set manually as MIME header — works identically |
| **Trade-off** | Sends from your personal Gmail. Google may flag high-volume programmatic sends and ask you to re-verify. |

**When to choose**: You want zero new accounts/services, already have a Gmail address, and subscriber count will stay under ~200.

### Gmail App Password setup

1. Go to [myaccount.google.com/security](https://myaccount.google.com/security)
2. Ensure **2-Step Verification** is enabled
3. Go to **App passwords** (search "app passwords" in account settings)
4. Create a new app password — name it "Headlines"
5. Copy the 16-character password
6. Set `GMAIL_APP_PASSWORD` in `.env` to this value

This is NOT your Google account password. It's a per-app credential that can be revoked independently.

### Why `aiosmtplib` instead of stdlib `smtplib`

The worker runs on asyncio (APScheduler + FastAPI). `smtplib` is synchronous — each send blocks the event loop for ~1 second (TLS handshake + transmission). For 50 subscribers, that's 50 seconds of total event loop blockage, starving the scheduler's other jobs (fetch, classify, dedup).

`aiosmtplib` is async-native. Each send yields back to the event loop during I/O waits. Other tasks interleave normally. It's the proper solution for async Python SMTP.

Note: The Resend SDK is also synchronous, but each HTTP call is fast (~100-300ms). The blocking is negligible. Gmail SMTP is slower per-call (~800ms-1.2s due to TLS handshake), making async essential.

### Scaling note: Gmail connection reuse

At MVP scale (<100 subscribers), opening a fresh SMTP connection per email is acceptable (~1s overhead each). If subscriber count grows past ~200, refactor to hold one connection open for the full batch:

```python
async with aiosmtplib.SMTP(hostname="smtp.gmail.com", port=587, start_tls=True) as smtp:
    await smtp.login(user, password)
    for subscriber in subscribers:
        await smtp.send_message(msg)  # reuses connection
```

This eliminates the per-email TLS handshake, dropping per-send time to ~50ms. Not needed for MVP.

## Database Schema

### Table: `subscribers`

| Column | Type | Constraints | Notes |
|--------|------|-------------|-------|
| `id` | `UUID` | PK, default `gen_random_uuid()` | |
| `email` | `TEXT` | NOT NULL, UNIQUE | Lowercase, trimmed |
| `token` | `TEXT` | NOT NULL, UNIQUE | HMAC-SHA256 for unsubscribe URLs |
| `status` | `TEXT` | NOT NULL, default `'active'` | `'active'` or `'unsubscribed'` |
| `subscribed_at` | `TIMESTAMPTZ` | NOT NULL | When they subscribed (or re-subscribed) |
| `unsubscribed_at` | `TIMESTAMPTZ` | nullable | When they unsubscribed |
| `created_at` | `TIMESTAMPTZ` | NOT NULL | TimestampMixin |
| `updated_at` | `TIMESTAMPTZ` | NOT NULL | TimestampMixin |

**Indexes:**
- `uq_subscribers_email` — unique on `email` (prevents duplicates, enables upsert)
- `uq_subscribers_token` — unique on `token` (fast unsubscribe lookup)
- `idx_subscribers_active` — partial index on `status` WHERE `status = 'active'` (fast subscriber list query)

**Token generation**: `hmac.new(SUBSCRIBE_SECRET.encode(), email.encode(), hashlib.sha256).hexdigest()`. Deterministic — same email always produces same token. No token accumulation on re-subscribe.

### Table: `email_send_logs`

| Column | Type | Constraints | Notes |
|--------|------|-------------|-------|
| `id` | `UUID` | PK, default `gen_random_uuid()` | |
| `subscriber_id` | `UUID` | FK → `subscribers.id`, NOT NULL | |
| `briefing_date` | `DATE` | NOT NULL | Which day's briefing was sent |
| `status` | `TEXT` | NOT NULL | `'sent'` or `'failed'` |
| `provider_id` | `TEXT` | nullable | Resend message ID or SMTP Message-ID |
| `error_message` | `TEXT` | nullable | Error details on failure |
| `sent_at` | `TIMESTAMPTZ` | NOT NULL, default `now()` | |

**Indexes:**
- `uq_email_send_logs_sub_date` — unique on `(subscriber_id, briefing_date)` — **idempotency guard**: prevents double-sends if the job reruns
- `idx_email_send_logs_date` — on `briefing_date` (admin queries)

## Backend Implementation

### Email sender abstraction: `backend/app/email/sender.py`

The single interface the worker uses. Follows the pattern of `workers/alerting.py` (no-op when not configured) and `llm/registry.py` (config-based dispatch).

```python
import logging
from dataclasses import dataclass

from app.config import settings

logger = logging.getLogger(__name__)


@dataclass
class EmailResult:
    message_id: str | None = None


def is_email_configured() -> bool:
    """Check if any email provider is configured."""
    return bool(settings.RESEND_API_KEY or (settings.GMAIL_USER and settings.GMAIL_APP_PASSWORD))


async def send_email(
    to: str,
    subject: str,
    html: str,
    headers: dict[str, str] | None = None,
) -> EmailResult:
    """Send an email using the configured provider. Returns result with message ID."""
    if settings.RESEND_API_KEY:
        return await _send_resend(to, subject, html, headers)
    if settings.GMAIL_USER and settings.GMAIL_APP_PASSWORD:
        return await _send_gmail(to, subject, html, headers)
    logger.debug("No email provider configured, skipping send")
    return EmailResult()


async def _send_resend(
    to: str, subject: str, html: str, headers: dict[str, str] | None,
) -> EmailResult:
    import resend

    resend.api_key = settings.RESEND_API_KEY
    response = resend.Emails.send({
        "from": settings.EMAIL_FROM,
        "to": to,
        "subject": subject,
        "html": html,
        "headers": headers or {},
    })
    msg_id = response.get("id") if isinstance(response, dict) else getattr(response, "id", None)
    return EmailResult(message_id=msg_id)


async def _send_gmail(
    to: str, subject: str, html: str, headers: dict[str, str] | None,
) -> EmailResult:
    import aiosmtplib
    from email.mime.multipart import MIMEMultipart
    from email.mime.text import MIMEText
    from email.utils import make_msgid

    msg = MIMEMultipart("alternative")
    msg["From"] = settings.EMAIL_FROM
    msg["To"] = to
    msg["Subject"] = subject
    msg["Message-ID"] = make_msgid(domain=settings.GMAIL_USER.split("@")[-1])
    if headers:
        for key, value in headers.items():
            msg[key] = value
    msg.attach(MIMEText(html, "html"))

    await aiosmtplib.send(
        msg,
        hostname="smtp.gmail.com",
        port=587,
        start_tls=True,
        username=settings.GMAIL_USER,
        password=settings.GMAIL_APP_PASSWORD,
    )
    return EmailResult(message_id=msg["Message-ID"])
```

### Router: `backend/app/routers/subscribe.py`

Three endpoints:

**`POST /api/subscribe`** — public, rate-limited

- Normalizes email: `email = body.email.strip().lower()`
- Generates token: `hmac.new(settings.SUBSCRIBE_SECRET.encode(), email.encode(), hashlib.sha256).hexdigest()`
- Upserts into `subscribers` using PostgreSQL `INSERT ... ON CONFLICT`:
  - If email is new → insert with status `active`
  - If email exists and `active` → no-op (idempotent)
  - If email exists and `unsubscribed` → set status back to `active`, update `subscribed_at`
- Returns `{"status": "subscribed"}`
- Rate limit: 5 requests per minute per IP (in-memory, see below)

```python
from collections import defaultdict
from time import monotonic

# In-memory rate limiter — single process, no Redis needed at this scale
_rate_buckets: dict[str, list[float]] = defaultdict(list)
RATE_LIMIT = 5
RATE_WINDOW = 60  # seconds

def _check_rate_limit(ip: str) -> bool:
    now = monotonic()
    bucket = _rate_buckets[ip]
    _rate_buckets[ip] = [t for t in bucket if now - t < RATE_WINDOW]
    if len(_rate_buckets[ip]) >= RATE_LIMIT:
        return False
    _rate_buckets[ip].append(now)
    return True
```

Applied as an early check in the endpoint, raises `HTTPException(429)` if exceeded.

**`GET /api/unsubscribe`** — public, returns HTML

- Accepts `token` query parameter
- Looks up subscriber by token
- If found and active: sets `status = 'unsubscribed'`, `unsubscribed_at = now()`
- Always returns a simple HTML confirmation page (even if token not found — don't leak info)
- Returns `HTMLResponse` directly, not JSON

```python
@router.get("/unsubscribe")
async def unsubscribe(token: str, db: AsyncSession = Depends(get_db)):
    result = await db.execute(select(Subscriber).where(Subscriber.token == token))
    subscriber = result.scalar_one_or_none()
    if subscriber and subscriber.status == "active":
        subscriber.status = "unsubscribed"
        subscriber.unsubscribed_at = datetime.now(timezone.utc)
    return HTMLResponse(UNSUBSCRIBE_HTML)
```

**`GET /api/admin/subscribers`** — admin-only (optional, for dashboard)

- Returns `{ "active": count, "unsubscribed": count }`
- Protected by `Depends(get_admin)` following existing admin router pattern

## Email Digest Worker

### File: `backend/app/workers/email_digest.py`

Follows the exact pattern of `_run_briefing` in `scheduler.py`: acquire session, query data, perform action, commit, log result. Uses the `email/sender.py` abstraction — never imports Resend or aiosmtplib directly.

```python
import logging
from datetime import datetime, timezone
from zoneinfo import ZoneInfo

from sqlalchemy import select

from app.config import settings
from app.database import async_session
from app.email.sender import is_email_configured, send_email
from app.models.briefing import Briefing
from app.models.email_send_log import EmailSendLog
from app.models.subscriber import Subscriber
from app.workers.email_template import render_digest_email

logger = logging.getLogger(__name__)


async def send_daily_digest():
    if not is_email_configured():
        return

    tz = ZoneInfo(settings.BRIEFING_TIMEZONE)
    today = datetime.now(tz).date()

    async with async_session() as db:
        try:
            # Get today's briefing
            result = await db.execute(
                select(Briefing).where(Briefing.type == "daily_news", Briefing.date == today)
            )
            briefing = result.scalar_one_or_none()
            if not briefing or not briefing.brief:
                logger.info("No briefing for %s, skipping digest", today)
                return

            # Get active subscribers
            result = await db.execute(
                select(Subscriber).where(Subscriber.status == "active")
            )
            subscribers = list(result.scalars().all())
            if not subscribers:
                logger.info("No active subscribers, skipping digest")
                return

            subject = f"Headlines — {today.strftime('%B %-d, %Y')}"
            sent = 0
            failed = 0

            for subscriber in subscribers:
                # Idempotency check
                existing = await db.execute(
                    select(EmailSendLog).where(
                        EmailSendLog.subscriber_id == subscriber.id,
                        EmailSendLog.briefing_date == today,
                    )
                )
                if existing.scalar_one_or_none():
                    continue

                unsubscribe_url = f"{settings.SITE_URL}/api/unsubscribe?token={subscriber.token}"
                html = render_digest_email(briefing, today, unsubscribe_url)
                headers = {
                    "List-Unsubscribe": f"<{unsubscribe_url}>",
                    "List-Unsubscribe-Post": "List-Unsubscribe=One-Click",
                }

                try:
                    result = await send_email(subscriber.email, subject, html, headers)
                    db.add(EmailSendLog(
                        subscriber_id=subscriber.id,
                        briefing_date=today,
                        status="sent",
                        provider_id=result.message_id,
                    ))
                    sent += 1
                except Exception as e:
                    db.add(EmailSendLog(
                        subscriber_id=subscriber.id,
                        briefing_date=today,
                        status="failed",
                        error_message=str(e)[:500],
                    ))
                    failed += 1

            await db.commit()
            logger.info("Daily digest for %s: sent=%d failed=%d total=%d", today, sent, failed, len(subscribers))

        except Exception as e:
            logger.error("Daily digest failed: %s", e)
            await db.rollback()
```

## Email Template

- **Header**: "Headlines" wordmark + date
- **The Brief**: The daily editorial summary (from `briefing.brief`)
- **In Focus** (conditional): Topic + analysis (from `briefing.focus_topic` / `briefing.focus_body`)
- **Footer**: Unsubscribe link


## Unsubscribe — CAN-SPAM Compliance

Every sent email includes:

1. **`List-Unsubscribe` header** — enables one-click unsubscribe in Gmail, Apple Mail, Outlook
2. **`List-Unsubscribe-Post` header** — RFC 8058 compliance for email client auto-unsubscribe
3. **Visible footer link** — "Unsubscribe" text link pointing to `GET /api/unsubscribe?token=xxx`

The unsubscribe endpoint:
- Sets `status = 'unsubscribed'` and `unsubscribed_at = now()`
- Returns a minimal HTML page: "You have been unsubscribed from Headlines."
- Is idempotent — clicking twice is harmless
- Does not leak whether the token exists (always shows the same page)

These headers work identically with both Resend and Gmail SMTP. Resend passes them as API parameters; Gmail sets them as MIME headers. The email client behavior is the same.

## Provider comparison at a glance

| | Resend | Gmail SMTP |
|---|---|---|
| **New account needed** | Yes (resend.com) | No |
| **Domain needed** | Yes (for custom From) | No |
| **DNS changes** | DKIM + SPF + DMARC | None |
| **Free email cap** | 100/day | 500/day (Gmail) or 2,000/day (Workspace) |
| **From address** | Any verified domain | Your Gmail address |
| **Setup time** | ~15 min (signup + DNS) | ~2 min (App Password) |
| **Deliverability risk** | Low (purpose-built) | Low-medium (Google may flag programmatic use) |
| **Best for** | Public-facing product | Personal use / small audience |

## What this plan does NOT include (Phase 2)

- **Double opt-in**: confirmation email flow, `pending` status, token expiration
- **Welcome email**: immediate send on subscribe
- **Digest preferences**: choose news, learning, or both
- **Batch sending**: Resend Batch API or Gmail connection reuse for 100+ subscribers
- **Admin subscriber dashboard**: list, search, export subscribers
- **Click tracking**: UTM parameters on links in the email
- **Bounce handling**: Resend webhooks or SMTP bounce parsing
- **Redis rate limiting**: for multi-replica API deployments
- **Email preview**: admin endpoint to preview today's digest before it sends
